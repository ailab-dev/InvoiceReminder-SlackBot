## 1. プロジェクト概要

### 1.1 目的

インターン生の毎月の給与（請求書）管理を自動化し、管理担当者（佐藤さん）の手作業負荷を削減する。
給与情報の確定と同時に請求書を自動発行し、インターン生・佐藤さん・Google Drive へ同時配布する。

### 1.2 背景・課題

- インターン生が毎月個別に給与情報を提出する必要があるが、フォーマットが統一されていない
- 提出漏れ・遅延が発生しやすい
- 佐藤さんが提出状況を個別に確認・催促する必要がある
- 振込漏れリスクがある
- 請求書のフォーマットが統一されておらず、インターン生が個別に作成している
- 請求書の保管・管理が煩雑

### 1.3 解決方針

- Slack Bot で毎月定期的に提出を案内
- 統一フォームで入力させ、不変情報は自動入力（変更も可）
- **確認はモーダル内の2ページ目で完結**（確認DMの廃止）
- 給与情報の確定と同時に請求書（PDF）を自動生成
- **PDF をインターン生DM・`#インターン給与` チャンネル・Google Drive の3箇所へ自動配布**
- **佐藤さんへの通知は都度DM廃止 → チャンネル投稿に統一**（通知量の最適化）
- 振込期限3日前に佐藤さんへサマリーDMを送り、振込完了ボタンで管理

## 3. 機能要件

### 3.1 月次チャンネル投稿（Bot による定期周知）

毎月1日に `#インターン給与` チャンネルにボットが投稿する。

```
┌──────────────────────────────────────────────┐
│  💰 今月の給与情報を提出してください            │
│                                              │
│  提出期限: 〇月末日（〇日後）                  │
│  ※前回登録した口座・氏名は自動で入力されます。  │
│                                              │
│  [ 📝 給与情報を提出する ]                    │
└──────────────────────────────────────────────┘
```

- インターン生の Slack ID は事前登録不要（逐一収集方式）
- 初回提出後は Redis にプロフィールを保存し、以後は自動入力

---

### 3.2 情報提出フロー（モーダル2ページ構成）

#### 3.2.1 提出モーダル（1ページ目）の入力項目

| フィールド | 種別 | 必須 | 初回 | 2回目以降 |
|---|---|---|---|---|
| 氏名 | テキスト入力 | ○ | 手入力 | 前回値を自動入力（変更可） |
| 住所 | テキスト入力 | ○ | 手入力 | 前回値を自動入力（変更可） |
| 電話番号 | テキスト入力 | ○ | 手入力 | 前回値を自動入力（変更可） |
| 対象年月 | テキスト（自動）| ○ | 今月を自動表示 | 今月を自動表示（固定） |
| 単価（円/時） | 数値入力 | ○ | 手入力 | 前回値を自動入力（変更可） |
| 稼働時間（時間） | 数値入力（小数可）| ○ | 手入力 | 手入力 |
| 経費1 費目名 | テキスト入力 | △ | 手入力（空可） | 前回の費目名を自動入力（変更可） |
| 経費1 金額（円） | 数値入力 | △ | 手入力（0可） | 手入力（毎回入力） |
| 経費2 費目名 | テキスト入力 | △ | 手入力（空可） | 前回の費目名を自動入力（変更可） |
| 経費2 金額（円） | 数値入力 | △ | 手入力（0可） | 手入力（毎回入力） |
| 経費3 費目名 | テキスト入力 | △ | 手入力（空可） | 前回の費目名を自動入力（変更可） |
| 経費3 金額（円） | 数値入力 | △ | 手入力（0可） | 手入力（毎回入力） |
| 振込口座 | テキスト入力 | ○ | 手入力 | 前回値を自動入力（変更可） |

> - 住所はプロフィールとして保存し、次回以降自動入力。請求書の請求者住所として使用。
> - 経費の費目名はプロフィールに「前回使った費目名」として保存し次回自動入力。金額は毎回手入力。
> - 経費行は最大3行。費目名・金額が両方空の行は無視する。費目名があれば金額は必須。
> - 行数の制限は Slack モーダルの仕様上、固定3行とする（動的追加は非対応）。

#### 3.2.2 確認画面（2ページ目）— モーダル内

1ページ目の「内容を確認する」ボタン押下後、**モーダルが2ページ目に切り替わる**（DM不要）。
Slack の `response_action: "push"` を使用し、金額計算済みの確認画面をモーダルスタック上に表示する。

```
┌──────────────────────────────────────────────────┐
│  📋 内容の確認                          ← 修正する│
│                                                  │
│  氏名: 鈴木 太郎                                  │
│  住所: 東京都〇〇区〇〇 1-2-3                      │
│  電話番号: 090-XXXX-XXXX                          │
│  対象月: 2026年4月                                │
│                                                  │
│  単価: ¥1,500/時                                 │
│  稼働時間: 40.0時間                               │
│  給与小計: ¥60,000                               │
│                                                  │
│  経費内訳:                                        │
│    移動費: ¥2,000                                │
│    交通費: ¥3,000                                │
│    AI利用費: ¥500                                │
│  経費合計: ¥5,500                                │
│  ─────────────────────────────────              │
│  税抜合計: ¥65,500                               │
│  消費税(10%): ¥6,550                             │
│  税込請求合計: ¥72,050                            │
│                                                  │
│  振込口座: 〇〇銀行 〇〇支店 普通 1234567           │
│  振込期限: 2026年5月31日                          │
│                                                  │
│             [ ✅ 確定して送信 ]                   │
└──────────────────────────────────────────────────┘
```

- **「✅ 確定して送信」**: モーダルを閉じ、PDF生成・Redis保存・各通知を実行
- **「← 修正する」（モーダルの戻るボタン）**: 1ページ目に戻り、入力値を引き継いで再編集

> **DM確認ステップは廃止。** 確認は完全にモーダル内で完結する。

#### 3.2.3 提出完了後の通知

**インターン生への通知（DM 1通のみ）**:
```
✅ 給与情報を提出しました（2026年4月分）

提出日時: 2026年4月30日 XX:XX
請求書番号: INV-2026-04-U12345678

📎 invoice_202604_鈴木太郎.pdf（添付）
```

**佐藤さんへの通知（`#インターン給与` チャンネルへの投稿）**:
```
📩 鈴木 太郎さんが給与情報を提出しました

対象月: 2026年4月
税込請求合計: ¥72,050
振込先: 〇〇銀行 〇〇支店 普通 1234567
振込期限: 2026年5月31日

📎 invoice_202604_鈴木太郎.pdf（添付）
```

> 都度DM（1人1通）は廃止。チャンネルを見れば提出状況と書類を一覧できる。
> 振込完了ボタンは振込サマリーDM（cron）にのみ設置する。

**Google Driveへの保存**:
- 指定フォルダに PDF を自動アップロード
- ファイル名: `invoice_{YYYY}{MM}_{intern_name}.pdf`

### 3.3 請求書（PDF）仕様

#### 3.3.1 発行タイミング

インターン生が「✅ 確定して送信」を押した瞬間にサーバーサイドで PDF を生成し、
Slack アップロード（インターン生DM・チャンネル）と Google Drive 保存を同時に実行する。

> **3秒制限への対処**: モーダルの `view_submission` には即座に 200 を返してモーダルを閉じ、
> PDF生成・アップロード処理は非同期で実行する。

#### 3.3.2 請求書レイアウト

```
─────────────────────────────────────────────────
                    請 求 書
─────────────────────────────────────────────────
発行日      : 2026年4月30日
請求書番号   : INV-2026-04-U12345678

請求先:
  株式会社AilaB 御中

請求者:
  鈴木 太郎
  東京都〇〇区〇〇 1-2-3
  090-XXXX-XXXX

─────────────────────────────────────────────────
件名: 2026年4月分 業務委託費

  稼働時間      : 40.0時間
  単価          : ¥1,500 / 時
  給与小計      : ¥60,000

  経費内訳:
    移動費      : ¥2,000
    交通費      : ¥3,000
    AI利用費    : ¥500
  経費合計      : ¥5,500

─────────────────────────────────────────────────
  税抜合計      : ¥65,500
  消費税（10%） : ¥6,550
  税込請求合計  : ¥72,050
─────────────────────────────────────────────────

お振込先:
  〇〇銀行 〇〇支店 普通 1234567
  口座名義: 鈴木 タロウ（※振込口座欄の文字列をそのまま記載）

振込期限: 2026年5月31日
─────────────────────────────────────────────────
```

#### 3.3.3 金額計算ルール

| 項目 | 計算式 |
|---|---|
| 給与小計 | `floor(単価 × 稼働時間)` |
| 経費合計 | `移動費 + 交通費 + AI利用費` |
| 税抜合計 | `給与小計 + 経費合計` |
| 消費税 | `round(税抜合計 × 0.10)` |
| 税込請求合計 | `税抜合計 + 消費税` |
| 振込期限 | 来月末日（例: 4月提出 → 5月31日） |

#### 3.3.4 請求書番号採番ルール

- フォーマット: `INV-{YYYY}-{MM}-{SlackUserID}`
- 例: `INV-2026-04-U12345678`

### 3.4 定期通知（Cron ジョブ）

毎朝9時（JST）に実行。

| タイミング | 処理 |
|---|---|
| 毎月1日 | `#インターン給与` に月次案内を投稿 |
| 月末3日前〜月末日 | 未提出の既知インターン生に DM でリマインド |
| 来月末3日前〜来月末日 | 佐藤さんに振込サマリーDM（提出一覧 + 振込完了ボタン + 未提出者リスト） |

**「既知インターン生」の定義**: 過去に1回以上提出したことがある Slack ユーザー（Redis に保存）

#### 3.4.1 インターン生へのリマインド DM

```

提出期限: 2026年4月30日（残り3日）
まだ提出されていません。下記から提出してください。

[ 📝 給与情報を提出する ]
```

#### 3.4.2 佐藤さんへの振込サマリー DM（振込完了ボタンはここのみ）

```
💰 振込期限が近づいています（2026年5月31日 残り3日）
```

### 3.5 振込完了処理

- 佐藤さんが振込サマリーDMの「✅ 振込完了」ボタンをクリック
- 対象メッセージを「振込完了済み」に書き換え
- Redis のステータスを `paid` に更新

```
☑️ 振込完了済み

氏名: 鈴木 太郎 / 2026年4月分
税込金額: ¥72,050
完了日: 2026年5月30日
```

## 4. 通知設計まとめ

現状と変更後の通知量を整理する。

| タイミング | 通知先 | 変更前 | 変更後 |
|---|---|---|---|
| 提出モーダル送信後（確認） | インターン生 | 確認DM送信 | ❌ 廃止（モーダル内で完結） |
| 提出確定後 | インターン生 | 確認DMをchat.updateで書き換え | ✅ DM 1通（PDF添付） |
| 提出確定後 | 佐藤さん | DM 1通（提出のたびに） | ✅ チャンネル投稿（PDF添付） |
| 提出確定後 | Google Drive | なし | ✅ PDF自動保存 |
| 月末3日前〜末日 | 未提出インターン生 | DM リマインド | 変更なし |
| 振込期限3日前〜末日 | 佐藤さん | DM サマリー（振込ボタン付き） | 変更なし |

**結果**: インターン生は提出時のDMが1通に削減。佐藤さんは都度DMがなくなりチャンネルで一覧管理。

## 5. 非機能要件

### 5.1 フェーズ1からの変更なし

- ホスティング: Vercel（Hobby プラン）
- スケジューラー: cron-job.org（既存の Cron ジョブを流用）

### 5.2 Slack レスポンスタイム

- Slack の `view_submission` は 3秒以内に応答必須
- PDF生成・Google Driveアップロードは応答後に非同期で実行
- インターン生への完了DMは若干の遅延が発生することがある（許容範囲）

### 5.3 テスト運用

テスト時は `MANAGER_SLACK_ID` を自分自身の Slack ID に設定し、
本番移行時に佐藤さんの Slack ID へ環境変数を切り替える。

## 7. データ設計

### 7.1 インターン生プロフィール（不変情報）

**Redis キー**: `intern:profile:{slack_user_id}`

```json
{
  "slack_user_id": "U12345678",
  "name": "鈴木 太郎",
  "address": "東京都〇〇区〇〇 1-2-3",
  "phone": "090-XXXX-XXXX",
  "bank_info": "〇〇銀行 〇〇支店 普通 1234567",
  "unit_price": 1500,
  "expense_names": ["移動費", "AI利用費", ""],
  "updated_at": "2026-04-01T10:00:00Z"
}
```

> `expense_names` は前回使用した費目名（最大3件）を保存し、次回のモーダルに自動入力する。

### 7.2 月次提出データ

**Redis キー**: `intern:salary:{id}`

```json
{
  "id": "uuid-v4",
  "intern_id": "U12345678",
  "intern_name": "鈴木 太郎",
  "intern_address": "東京都〇〇区〇〇 1-2-3",
  "month": "2026-04",
  "unit_price": 1500,
  "working_hours": 40.0,
  "expenses": [
    { "name": "移動費", "amount": 2000 },
    { "name": "AI利用費", "amount": 500 }
  ],
  "total_salary": 60000,
  "total_expense": 2500,
  "subtotal": 62500,
  "tax_amount": 6250,
  "total_amount": 68750,
  "bank_info": "〇〇銀行 〇〇支店 普通 1234567",
  "payment_due": "2026-05-31",
  "invoice_number": "INV-2026-04-U12345678",
  "status": "submitted",
  "submitted_at": "2026-04-28T10:00:00Z",
  "paid_at": null
}
```

> - `expenses` は `{name: string, amount: number}` の配列（最大3件、空行は含まない）
> - `subtotal`（税抜合計）、`tax_amount`（消費税）、`total_amount`（税込合計）を明示的に分離して保存する。

### 7.3 Redis キー一覧

| キー | 型 | 内容 |
|---|---|---|
| `intern:profile:{slack_user_id}` | String (JSON) | インターン生のプロフィール |
| `intern:known_users` | Set | 過去に提出経験のある Slack User ID |
| `intern:salary:{id}` | String (JSON) | 月次提出データ本体 |
| `intern:salary:months:{YYYY-MM}` | Set | その月の提出 ID 集合 |
| `intern:salary:submitted_users:{YYYY-MM}` | Set | その月に提出済みの User ID 集合 |

> **Pending データは廃止**: モーダル2ページ構成により、`intern:salary:pending:{user_id}` キーは不要になる。
> 確認データは Slack の `private_metadata`（モーダルに紐付けたJSON）に保持する。

### 7.4 ステータス遷移

```
[入力中・モーダル1ページ目]
  → (「内容を確認する」ボタン)
[確認・モーダル2ページ目]
  → (「← 修正する」) → [入力中に戻る]
  → (「✅ 確定して送信」) → submitted
                         → (「✅ 振込完了」) → paid
```
