## 1. プロジェクト概要

### 1.1 目的

インターン生の毎月の給与（請求書）管理を自動化し、管理担当者（佐藤さん）の手作業負荷を削減する。

### 1.2 背景・課題

- インターン生が毎月個別に給与情報を提出する必要があるが、フォーマットが統一されていない
- 提出漏れ・遅延が発生しやすい
- 佐藤さんが提出状況を個別に確認・催促する必要がある
- 振込漏れリスクがある

### 1.3 解決方針

- Slack Bot で毎月定期的に提出を案内
- 統一フォームで入力させ、不変情報は自動入力（変更も可）
- 提出前にインターン生本人が確認するワンクッションを設ける
- 佐藤さんへ提出状況をまとめて通知し、振込完了ボタンで管理

## 3. 機能要件

### 3.1 月次チャンネル投稿（Bot による定期周知）

毎月1日に専用チャンネル（例: `#インターン給与`）にボットが以下を投稿する。

```
┌──────────────────────────────────────────────┐
│  💰 今月の給与情報を提出してください            │
│                                              │
│  提出期限: 〇月末日（〇日後）                  │
│  ※前回登録した口座・氏名は自動で入力されます。  │
│                                              │
│  [ 📝 給与情報を提出する ]                    │
└──────────────────────────────────────────────┘
```

- インターン生がこのボタンをクリックしてモーダルを開く
- インターン生の Slack ID は事前登録不要（逐一収集方式）
- 初回提出後は Redis にプロフィールを保存し、以後は自動入力

---

### 3.2 情報提出フロー

#### 3.2.1 提出モーダルの入力項目

| フィールド | 種別 | 必須 | 初回 | 2回目以降 |
|---|---|---|---|---|
| 氏名 | テキスト入力 | ○ | 手入力 | 前回値を自動入力（変更可） |
| 電話番号 | テキスト入力 | ○ | 手入力 | 前回値を自動入力（変更可） |
| 対象年月 | テキスト（自動）| ○ | 今月を自動表示 | 今月を自動表示（固定） |
| 単価（円/時） | 数値入力 | ○ | 手入力 | 前回値を自動入力（変更可） |
| 稼働時間（時間） | 数値入力（小数可）| ○ | 手入力 | 手入力 |
| 移動費（円） | 数値入力 | △ | 手入力（0可） | 手入力（0可） |
| 交通費（円） | 数値入力 | △ | 手入力（0可） | 手入力（0可） |
| AI利用費（円） | 数値入力 | △ | 手入力（0可） | 手入力（0可） |
| 振込口座 | テキスト入力 | ○ | 手入力 | 前回値を自動入力（変更可） |

**振込口座の入力フォーマット例**: `〇〇銀行　〇〇支店　普通　1234567`

#### 3.2.2 提出前確認（インターン生への確認ステップ）

モーダル送信後、インターン生の DM に確認メッセージを送信する。
佐藤さんへの送信は確認後に行う。

```
以下の内容で佐藤さんに送りますか？

氏名: 鈴木 太郎
電話番号: 090-XXXX-XXXX
対象月: 2026年4月
単価: ¥1,500/時
稼働時間: 40.0時間
給与小計: ¥60,000
経費内訳:
  移動費: ¥2,000
  交通費: ¥3,000
  AI利用費: ¥500
経費合計: ¥5,500
┄┄┄┄┄┄┄┄┄┄
請求合計: ¥65,500
振込口座: 〇〇銀行 〇〇支店 普通 1234567

[ ✅ 確定して送信 ]   [ ✏️ 修正する ]
```

- **「✅ 確定して送信」**: 佐藤さんへ通知し、Redisへ保存
- **「✏️ 修正する」**: モーダルを再度開き、同じ内容を編集状態で表示

#### 3.2.3 提出完了後の通知

インターン生へ:
```
✅ 給与情報を提出しました（2026年4月分）
提出日時: 2026年4月30日 XX:XX
```

佐藤さんへ（リアルタイム通知）:
```
📩 給与情報が提出されました

氏名: 鈴木 太郎
対象月: 2026年4月
請求合計: ¥65,500
  内訳: 単価¥1,500 × 40.0h + 経費¥5,500
振込先: 〇〇銀行 〇〇支店 普通 1234567
振込期限: 2026年5月31日

[ ✅ 振込完了 ]
```

---

### 3.3 定期通知（Cron ジョブ）

毎朝9時（JST）に実行。日付に応じて以下のロジックを動作させる。

| タイミング | 処理 |
|---|---|
| 毎月1日 | 専用チャンネルに月次案内を投稿（ピン留め更新） |
| 月末3日前〜月末日 | 未提出の既知インターン生に DM でリマインド |
| 来月末3日前〜来月末日 | 佐藤さんに振込サマリー DM（提出一覧 + 未提出者リスト） |

**「既知インターン生」の定義**: 過去に1回以上提出したことがある Slack ユーザー（Redis に保存）

#### 3.3.1 インターン生へのリマインド DM

```
⚠️ 給与情報の提出期限が近づいています

提出期限: 2026年4月30日（残り3日）

まだ提出されていません。
下記から提出してください。

[ 📝 給与情報を提出する ]
```

#### 3.3.2 佐藤さんへの振込サマリー DM

```
💰 振込期限が近づいています（2026年5月31日 残り3日）

■ 提出済み（3名）
─────────────────────────────
鈴木 太郎   ¥65,500   〇〇銀行 〇〇支店 普通 1234567  [ ✅ 振込完了 ]
山田 花子   ¥52,000   △△銀行 △△支店 普通 9876543  [ ✅ 振込完了 ]
田中 健   ¥48,000   □□銀行 □□支店 普通 5555555  [ ✅ 振込完了 ]

■ 未提出（1名）
─────────────────────────────
佐藤 次郎（未提出）
```

---

### 3.4 振込完了処理

- 佐藤さんが「✅ 振込完了」ボタンをクリック
- 対象のリマインダーメッセージを「振込完了済み」に書き換え
- Redis のステータスを `paid` に更新

```
☑️ 振込完了済み

氏名: 鈴木 太郎 / 2026年4月分
金額: ¥65,500
完了日: 2026年5月30日
```

## 6. データ設計

### 6.1 インターン生プロフィール（不変情報）

**Redis キー**: `intern:profile:{slack_user_id}`
**型**: String (JSON)

```json
{
  "slack_user_id": "U12345678",
  "name": "鈴木 太郎",
  "phone": "090-XXXX-XXXX",
  "bank_info": "〇〇銀行 〇〇支店 普通 1234567",
  "unit_price": 1500,
  "updated_at": "2026-04-01T10:00:00Z"
}
```

### 6.2 月次提出データ

**Redis キー**: `intern:salary:{id}`
**型**: String (JSON)

```json
{
  "id": "uuid-v4",
  "intern_id": "U12345678",
  "intern_name": "鈴木 太郎",
  "month": "2026-04",
  "unit_price": 1500,
  "working_hours": 40.0,
  "expense_transport": 2000,
  "expense_travel": 3000,
  "expense_ai": 500,
  "total_salary": 60000,
  "total_expense": 5500,
  "total_amount": 65500,
  "bank_info": "〇〇銀行 〇〇支店 普通 1234567",
  "payment_due": "2026-05-31",
  "status": "submitted",
  "submitted_at": "2026-04-28T10:00:00Z",
  "paid_at": null
}
```

### 6.3 Redis キー一覧

| キー | 型 | 内容 |
|---|---|---|
| `intern:profile:{slack_user_id}` | String (JSON) | インターン生のプロフィール（不変情報） |
| `intern:known_users` | Set | 過去に1回以上提出したことがある Slack User ID の集合 |
| `intern:salary:{id}` | String (JSON) | 月次提出データ本体 |
| `intern:salary:months:{YYYY-MM}` | Set | その月の提出 ID 集合 |
| `intern:salary:submitted_users:{YYYY-MM}` | Set | その月に提出済みの Slack User ID 集合 |
| `intern:salary:pending:{slack_user_id}` | String (JSON) | 確認待ちの一時データ（確定/修正されたら削除） |

### 6.4 ステータス遷移

```
[入力中] → (確定ボタン) → submitted → (振込完了ボタン) → paid
         → (修正ボタン) → [入力中に戻る]
```

## 8. API エンドポイント設計

### 8.1 新規エンドポイント

| エンドポイント | 役割 |
|---|---|
| `POST /api/cron/intern-notify` | 日次 Cron: インターン生へのリマインド・佐藤さんへのサマリー通知 |

### 8.2 既存エンドポイントへの追加

**`POST /api/slack/interactions`** に以下を追記:

| type | action_id / callback_id | 処理 |
|---|---|---|
| `block_actions` | `open_intern_salary_modal` | 給与入力モーダルを開く（プロフィール自動入力） |
| `view_submission` | `intern_salary_modal` | 入力値を一時保存 → インターン生に確認 DM |
| `block_actions` | `confirm_intern_salary` | 確定: Redis保存 + 佐藤さんへ通知 |
| `block_actions` | `edit_intern_salary` | 修正: モーダルを再表示（入力値を引き継ぎ） |
| `block_actions` | `mark_intern_paid` | 振込完了: ステータス更新 + メッセージ書き換え |

## 9. UX フロー

```
[インターン生]
  │
  ├─ 毎月1日: チャンネルに案内投稿を見る
  │  または
  ├─ 月末3日前: DM リマインドを受け取る
  │
  ▼
「📝 給与情報を提出する」ボタンをクリック
  │
  ▼
モーダルが開く（氏名・電話・単価・口座は自動入力済み）
  │
  ├─ 今月の稼働時間・経費を入力して「送信」
  │
  ▼
確認 DM を受け取る（内容・金額サマリー表示）
  │
  ├─「✅ 確定して送信」→ 佐藤さんに通知送信・完了DM
  └─「✏️ 修正する」→ モーダルを再表示

[佐藤さん]
  │
  ├─ 提出があるたびにリアルタイムで DM 通知
  │
  ├─ 振込期限3日前: サマリー DM（提出一覧 + ✅ボタン）
  │
  └─「✅ 振込完了」をクリック → 完了処理
```
